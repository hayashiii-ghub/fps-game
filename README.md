# KIMI GROK FPS

ブラウザで遊べる FPS。three.js 製・ビルド不要・オフライン動作。
マップは砂漠（DESERT）と密林（JUNGLE）の2つ（今後も追加予定）。

**▶ プレイ: https://hayashiii-ghub.github.io/fps-game/**

## 遊び方

### 起動

- 上記 GitHub Pages の URL、または
- ローカルでは `index.html` を開く（three.js は同梱済み・オフライン可）
- うまく動かない場合はプロジェクト直下で `python3 -m http.server 8765` を起動し、`http://127.0.0.1:8765/` にアクセス

### 目的 / モード

ロビーで武器・マップ・モードを選ぶ（カードをクリック。マップ選択は背景に即反映）。

- **MAIN / SUB WEAPON** — アサルト / SMG / ショットガン / スナイパーから重複なしで2つ選択。ハンドガンは全員が常備（特殊枠）
  - SMG: 高連射・軽快だが遠距離は苦手（近中距離）
  - ショットガン: 8ペレット散弾・ポンプアクション。近距離決戦用
- **MAP: DESERT** — 砂漠の拠点。開けた射線とコンテナ群。遠距離の撃ち合いが主体
- **MAP: JUNGLE** — 密林（PUBG Sanhok 参考）。中央遺跡・東リゾート・南港・西採石場・北西訓練場・北東岩窟。茂みは弾と視線を遮るが通り抜けられる

モード:

- **SURVIVAL** — 5ステージ防衛（接触→狙撃線→強襲→砂嵐→最終防衛）。難易度は段階的に上がり、Stage 5 が山場。自動回復あり。
  - Stage 2: 敵狙撃兵撃破でスナイパーライフル入手（ロードアウトで所持済みなら狙撃弾に変換）
  - Stage 3: 精鋭撃破（またはクリア時保証）で強化防具（被ダメ約28%軽減）
  - ステージクリア後は補給タイム（中央に弾薬・グレ・キットが落ちる。定期湧きはなし）
- **TEAM DEATHMATCH** — 5v5・5分・キル数勝負。装備はロビーのロードアウト（開始グレ2・キット2）。撃破ドロップあり。自動回復なし。
  - マップ中央に補給箱（約42秒周期）。弾薬・グレ・キットが各×2、たまに強化防具も出る

### 操作

| キー | 動作 |
| --- | --- |
| `W A S D` | 移動 |
| `Shift` | ダッシュ（射撃・ADS不可） |
| `C` | しゃがみ |
| `C + Shift` | スコープ（スナイパー時。そのとき Shift はスプリントにしない） |
| `Space` | ジャンプ |
| 左クリック | 射撃 / グレネード投擲（構え中） |
| 右クリック | 照準（アサルト・スナイパーは ADS。ハンドガンは腰撃ちのまま減速＋レティクル縮小） |
| `R` | リロード |
| `G` | グレネード構え / 解除（軌道を見てクリック投擲） |
| `F` | 応急キット使用（約2秒。所持して任意タイミング） |
| `Q` / `E` | 所持武器を循環切替（向きだけ逆） |
| `ESC` | ポーズ（再開 / ロビーへ） |

所持上限: グレネード **5** / 応急キット **3**。

### デバッグモード

URLにクエリを付けると検証用モードで起動する。

- `index.html?debug=1` : ポインタロックなしで即開始・状態ログ出力
- `&mode=tdm` : TDM で開始（debug時）
- `&map=jungle` : JUNGLE マップで開始（debug時）
- `&main=smg&sub=shotgun` : ロードアウト指定（debug時）
- `&shoot=1` : 自動で照準・射撃するボット
- `&fast=N` : 1フレームでNステップ進める時間加速シミュレーション
- `&ads=1` : ADS状態で開始

## 主なシステム

### ガンプレイ

- アサルト / SMG / ショットガン / ハンドガン / スナイパー（ロビーでメイン＋サブを選択。ハンドガンは常備）
- リコイル・ブルーム・ADS・ボルトアクション（スナイパーは射撃後のボルト中にスコープ不可）・ポンプアクション（ショットガンは8ペレット散弾）
- 部位判定（ヘッド／胴／四肢）。スナイパーは頭一撃・胴は非一撃
- グレネードは構え＋軌道表示からの投擲。所持上限あり

### 敵AI

- パトロール → 視認 or 銃声を聞きつけて索敵 → 交戦のステートマシン
- バースト射撃、ストレイフ、間合い管理、グレネード回避
- 被弾すると遮蔽へ退避。近くの味方が回り込み
- 足音・リロード音は距離減衰＋パン付き

### マップ / HUD

- 2マップ制（`world.js` の `MAP_DEFS` / `buildMap` で切替。マップ固有物は `mapGroup` 配下にまとめ、切替時にコライダ・弾判定メッシュをクリアして再構築）
- 移動衝突は Y 回転 OBB（建物・コンテナなどは見た目一致）。JUNGLE の木は幹のみ OBB、茂みは移動判定なし（弾・敵の視線は遮る）
- 左上ミニマップ（障害物・自機・味方。敵は非表示）
- スナイパースコープ中も体力・弾薬・マップを表示
- リザルトにキル数・HS率・最長キル距離・グレキルなどを表示

### その他

- 全音響を Web Audio API で手続き合成（銃声はクラック/ブラスト/低音/反響尾の4層・武器別音色、ボルト・ポンプ・リロードの金属共鳴、距離減衰エコー、足音・風・遠雷・ヒット音）
- ロビーはシネマティック仕様（低空ドリー映像・レターボックス・スキャンライン・コーナーブラケット・出撃フェード）
- 敵ドロップ（弾薬 / キット / グレ）。上限時は拾えず、MAX表示は1回だけ
- キルフィード、死亡カメラ、ポーズからロビーへ戻れる

## 制作の流れ

1. **要件整理** — 「ブラウザですぐ遊べる」「リアル志向」「ビルド不要」を優先し、three.js 同梱の静的ファイル構成に決定
2. **ワールド生成** — Canvas 2D 手続きテクスチャと拠点レイアウト。移動は OBB、弾は葉メッシュ単位で判定
3. **ガンプレイ** — リコイル／スプレッド／ADS／リロード／トレーサー／グレネード構え投げ／キット所持使用
4. **敵AI** — patrol / search / combat。視線レイキャスト、聴覚索敵、退避・回り込み
5. **モード** — Survival 5ステージ＋報酬進行、TDM 5v5＋中央補給争奪
6. **音響・HUD** — 手続き音、ミニマップ、スコープオーバーレイ、リザルト統計
7. **バランス・衝突の見直し** — パッチ積み上げより単純な当たり設計へ回帰しつつ手触りを調整
8. **マップ拡張** — `MAP_DEFS` による切替基盤化と JUNGLE（Sanhok 参考）追加。茂みは弾・視線を遮り移動は通過する隠れ場所
9. **武器・音響・ロビー強化** — SMG/ショットガン追加とロードアウト選択、銃声の多層化、シネマティックロビー

## 使用した技術

- **three.js r128** — WebGL、PCFSoftシャドウ、指数フォグ
- **Canvas 2D 手続きテクスチャ** — 画像素材ゼロで砂・金属・迷彩などを生成
- **Web Audio API 手続きサウンド** — 音声ファイルゼロで銃声・足音・環境音を合成
- **Pointer Lock API** — FPS のマウスルック
- **レイキャスト** — 弾丸着弾・敵の視線（LOS）
- **Y 回転 OBB** — 移動衝突（壁スライドなし・押し出し＋安全位置ロールバック）
- **オブジェクトプーリング** — トレーサー・パーティクル・薬莢の GC 抑制
